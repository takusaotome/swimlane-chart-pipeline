# Process Analysis Checklist

requirements.md の出力前に、以下の全項目を確認してください。

## 必須項目 (Must Have)

- [ ] **プロセス名**が定義されている
- [ ] **目的**が1-2文で記述されている
- [ ] **トリガー条件**（何がプロセスを開始するか）が明確
- [ ] **頻度**（毎日/毎週/毎月/随時等）が記載されている
- [ ] **関連部門が2つ以上**列挙されている（1部門ならスイムレーン不要）
- [ ] **各部門の役割**が説明されている
- [ ] **プロセスステップが3つ以上**ある
- [ ] **各ステップの担当部門**が割り当てられている
- [ ] **開始ノード**と**終了ノード**に相当するステップがある
- [ ] **全判断ポイントの両分岐**（Yes/No）が記載されている

## 推奨項目 (Should Have)

- [ ] **使用システム・ツール**が各ステップで特定されている
- [ ] **入力/出力**が各ステップで記載されている
- [ ] **工程フェーズ候補**（columns）が列挙されている
- [ ] **部門間のハンドオフ**が明示されている
- [ ] **例外パス**が少なくとも1つ記載されている

## あれば望ましい (Nice to Have)

- [ ] **SLA・所要時間**の目安がある
- [ ] **承認フロー**の詳細がある
- [ ] **並行処理**のパターンが特定されている
- [ ] **エスカレーション先**が定義されている

## チャート化可能性の判定

### Ready（チャート化可能）
- 必須項目が全て ✅
- 推奨項目が概ね ✅

### Needs Revision（修正が必要）
- 必須項目に1-2個の ❌ がある
- 推奨項目に複数の ❌ がある

### Insufficient（情報不足）
- 必須項目に3個以上の ❌ がある
- 基本的な部門情報やプロセスフローが不明確
