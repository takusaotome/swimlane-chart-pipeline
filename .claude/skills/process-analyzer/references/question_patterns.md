# Question Patterns

AskUserQuestion ツールで質問する際のパターン集。
質問はカテゴリ別に優先度順で整理されています。

## 優先度1: 部門・役職の特定

| 質問パターン | 使用場面 |
|---|---|
| 「[作業X]を実行する担当部門はどこですか？」 | 担当部門が不明 |
| 「[部門A]と[部門B]は別のスイムレーンにすべきですか？」 | 部門の粒度が不明 |
| 「このプロセスに関与する外部ステークホルダーはいますか？」 | 外部関与者の確認 |
| 「承認者は[部門X]の誰ですか？（部長/課長等）」 | 承認者の粒度 |

## 優先度2: 判断ポイントの分岐条件

| 質問パターン | 使用場面 |
|---|---|
| 「[判断X]で"No"の場合、どのステップに進みますか？」 | 分岐先が不明 |
| 「[判断X]の具体的な条件は何ですか？（閾値等）」 | 条件が曖昧 |
| 「[判断X]で判断に迷った場合のエスカレーション先は？」 | 例外パス確認 |

## 優先度3: 使用システム・ツール

| 質問パターン | 使用場面 |
|---|---|
| 「[作業X]で使用するシステムは何ですか？」 | システム未特定 |
| 「[システムA]から[システムB]へのデータ移動は手動ですか？自動ですか？」 | 連携方法 |
| 「通知・連絡手段はメール/Slack/電話のどれですか？」 | コミュニケーション手段 |

## 優先度4: 工程フェーズ・タイムライン

| 質問パターン | 使用場面 |
|---|---|
| 「このプロセス全体の所要時間はどのくらいですか？」 | 全体の時間感 |
| 「[ステップA]と[ステップB]の間に待ち時間はありますか？」 | フェーズの区切り |
| 「日付・時刻ベースのマイルストーンはありますか？」 | カラムの候補 |

## 優先度5: 例外パス

| 質問パターン | 使用場面 |
|---|---|
| 「差し戻しが発生するケースはありますか？」 | ループ検出 |
| 「途中でプロセスが中止になるケースはありますか？」 | 中断パス |
| 「期限切れの場合はどうなりますか？」 | タイムアウト処理 |

## AskUserQuestion の使い方ガイドライン

### 1ラウンドあたりの質問数
- 最大4問（AskUserQuestion の制約）
- 関連する質問をグループ化して効率的に

### 質問の書き方
- 具体的に（「この辺り」ではなく「Step 3 と Step 4 の間」）
- 選択肢を提供（可能な場合）
- なぜ重要かを簡潔に説明

### 回答が得られなかった場合
- 合理的なデフォルトを (Assumption) として記録
- 「前提・未確認事項」セクションに記載
